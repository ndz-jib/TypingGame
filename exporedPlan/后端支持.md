# 打字游戏 - 后端 API 文档与前端对接指南

## 项目概述

一款本地单人打字游戏桌面应用，基于 **Electron + Vue 3** 构建，后端使用 **Python Flask** 提供 API 服务。

---

## 技术架构

| 层级 | 技术 | 说明 |
|------|------|------|
| 桌面框架 | Electron | 主进程 + 渲染进程 |
| 前端框架 | Vue 3 + Vite + Pinia | 渲染进程 |
| 后端框架 | Python Flask | 本地 HTTP 服务（端口 5000） |
| 数据存储 | JSON 文件 | 本地 `./data/` 目录 |

## 前端资源访问规则（重要）

### 核心原则
**静态资源（图片、字体、音效）直接通过本地文件路径访问，不走 HTTP。**

### 原因
- 这些文件存储在 `./data/` 目录下
- Electron 渲染进程可以直接访问本地文件系统
- 避免 HTTP 传输开销，简化前端逻辑

### 资源访问路径对照表

| 资源类型 | 存储位置 | 前端访问路径 |
|---------|---------|-------------|
| 头像 | `./data/picture/avatar.png` | `./data/picture/avatar.png` |
| 字体 | `./data/font/default.ttf` | `./data/font/default.ttf` |
| 按键音效 | `./data/voice/keypress.mp3` | `./data/voice/keypress.mp3` |
| 错误音效 | `./data/voice/error.mp3` | `./data/voice/error.mp3` |
| 完成音效 | `./data/voice/complete.mp3` | `./data/voice/complete.mp3` |
| 文章文件 | `./data/txt/*.txt` | **不走文件路径，通过 API 获取内容** |
| 配置文件 | `./data/json/*.json` | **不走文件路径，通过 API 读写** |

### 为什么文章和配置不能直接读文件？
- 文章需要后端分词、拆分大文件逻辑
- 配置需要后端校验、默认值合并
- 统一走 API 保证数据一致性

## 前端访问方式总结

| 内容 | 访问方式 | 示例 |
|------|---------|------|
| 头像图片 | 本地路径 | `<img src="./data/picture/avatar.png">` |
| 字体文件 | CSS 本地路径 | `@font-face { src: url('./data/font/default.ttf') }` |
| 音效文件 | 本地路径 | `new Audio('./data/voice/keypress.mp3')` |
| 文章内容 | HTTP API | `GET /api/articles/xxx.txt` |
| 单词表 | HTTP API | `GET /api/vocabulary` |
| 错词表 | HTTP API | `GET /api/mistake` |
| 配置 | HTTP API | `GET /api/config` |
| 上传文件 | HTTP API | `POST /api/upload/avatar`（multipart） |
| 导入导出 | HTTP API | `POST /api/export/data` |


## 启动方式

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

```bash
python serve.py
```

### 3. 前端开发

```bash
cd frontend
npm install
```


## 前后端同步策略

| 存储位置 | 内容 | 更新时机 |
|---------|------|---------|
| 前端 Store | 游戏进度（本次会话） | 实时 |
| 前端 Store | 本次错误单词列表 | 实时 |
| 后端 JSON | 玩家信息、积分、统计 | 退出/完成时 |
| 后端 JSON | 错词表（历史累积） | 退出/完成时（调用 `/mistake/record`） |

**核心原则**：前端 Store 为主，后端仅持久化最终结果。

## 注意事项

1. **静态资源路径**：前端直接使用 `./data/xxx`，无需拼接 HTTP
2. **文件上传**：使用 `multipart/form-data`，字段名为 `file`
3. **文章导入**：大文件自动拆分，前端需处理拆分后的多个文件
4. **错词记录**：调用 `/mistake/record` 时会自动检查单词表，不存在则自动添加
5. **数据导入**：返回 `backupPath`，前端可提示用户重启应用
6. **字体加载**：上传后需要前端重新加载 CSS 或刷新页面

## 常见问题

**Q: 头像上传后不立即显示？**
A: 上传成功后会覆盖 `avatar.png`，前端需要重新加载图片（加时间戳刷新缓存）。

**Q: 字体上传后不生效？**
A: 字体需要重新加载 CSS 或重启应用。

**Q: 文章导入后单词表没有更新？**
A: 检查分词规则：单词必须包含字母，纯数字/符号会被过滤。

**Q: 如何获取已导入的文章列表？**
A: `GET /api/articles` 返回所有 `.txt` 文件信息。
